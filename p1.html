Gestión de USB booteable, dispositivos físicos y archivos importantes.

1. Instalación de un sistema Linux en USB

Objetivo USB booteable
  1. Analizar qué sistemas operativos pueden ser ideales para la instalación.
  2. Investigar las herramientas para crear un USB booteable.
    Yo usé Rufus en Windows.
  3. Probar que el USB booteable funciona (relacionado con el punto 2).
  4. Documentar de manera breve las decisiones tomadas.
    Instalas Rufus, conectas el USB, lo transformas en un bootable y le metes la ISO deseada. Luego seleccionas live boot e inicias sesión en la distro de Linux que hayas puesto en el USB.

2. Ejecutar el USB en los ordenadores del grupo

Prueba acceso y discos
  1. Probar sin instalar el sistema operativo; comprobar que se puede interactuar con herramientas y aplicaciones.
    1. Puede ser necesario modificar ajustes desde la BIOS (orden de arranque).
  2. Analizar los discos e información que hay en el ordenador (discos de Windows).
  3. ¿Se puede ver la información sensible de los usuarios de Windows desde el sistema operativo del USB?
  4. ¿Se puede modificar la información de los usuarios de Windows desde el sistema operativo del USB?
  5. ¿Hay algún caso exitoso en el grupo donde las dos respuestas anteriores sean similares? ¿Algún caso en el que las respuestas sean no? ¿Qué diferencias existen en los ordenadores del grupo?
    En Windows 11, por ejemplo, existe un programa llamado BitLocker que no te deja ver ni modificar datos. Cuando intenté hacer dual boot de Windows / Ubuntu, tuve que desactivarlo con la clave de activación de la licencia de Windows.

3. Preguntas extra

Investigación Windows
  1. ¿Qué archivo puedo modificar (no lo hagáis, solo investigad) para poder eliminar la contraseña y una vez que inicie sesión en Windows lo pueda hacer sin la necesidad de la contraseña?
    SAM (Security Account Manager)
  2. ¿Existe algún archivo que se pueda ejecutar siempre en el arranque en Windows? Si existe, ¿cómo se llama?
    Carpeta de Inicio (Startup)

4. Valoración de la práctica

Reflexión utilidad y dificultad
  1. Valora de manera subjetiva la utilidad real de la práctica.
    1. ¿Qué se puede hacer con lo aprendido?
      Editar archivos privados en un ordenador.
    2. ¿Cómo se puede uno proteger?
      Activando sistemas de seguridad como BitLocker.
  2. Valora la dificultad de la práctica (con intención de ajustar prácticas futuras).
    No me ha parecido difícil; está bien para aprender a hacer Live boot y lo que implica a nivel de seguridad.
